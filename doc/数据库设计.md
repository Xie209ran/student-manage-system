# 教学数字平台后台管理系统 - 数据库设计文档

## 1. 表关系说明

### 1.1 实体关系图（简化版）

```
用户表 (sys_user)
    ↓ 1:N
班级表 (class) ← 负责老师关联用户表
    ↓ 1:N
学生表 (student)
    ↓ 1:N
    ├─→ 考勤记录表 (attendance)
    ├─→ 成绩记录表 (score)
    └─→ 作业提交表 (homework_submission)
    
课程表 (course_schedule)
    ↓ 关联班级表和用户表（任课教师）
    
作业表 (homework)
    ↓ 1:N
    └─→ 作业提交表 (homework_submission)
```

### 1.2 表关系详细说明

| 主表 | 从表 | 关联字段 | 关系类型 | 说明 |
|-----|------|---------|---------|------|
| sys_user（用户表） | class（班级表） | class.teacher_id → sys_user.id | 1:N | 一个用户可以负责多个班级 |
| class（班级表） | student（学生表） | student.class_id → class.id | 1:N | 一个班级可以有多个学生，一个学生只能属于一个班级 |
| student（学生表） | attendance（考勤记录表） | attendance.student_id → student.id | 1:N | 一个学生可以有多条考勤记录 |
| student（学生表） | score（成绩记录表） | score.student_id → student.id | 1:N | 一个学生可以有多条成绩记录 |
| student（学生表） | homework_submission（作业提交表） | homework_submission.student_id → student.id | 1:N | 一个学生可以提交多次作业 |
| class（班级表） | course_schedule（课程表） | course_schedule.class_id → class.id | 1:N | 一个班级可以有多条课程安排 |
| sys_user（用户表） | course_schedule（课程表） | course_schedule.teacher_id → sys_user.id | 1:N | 一个教师可以教授多门课程 |
| class（班级表） | homework（作业表） | homework.class_id → class.id | 1:N | 一个班级可以接收多个作业 |
| sys_user（用户表） | homework（作业表） | homework.teacher_id → sys_user.id | 1:N | 一个教师可以发布多个作业 |
| homework（作业表） | homework_submission（作业提交表） | homework_submission.homework_id → homework.id | 1:N | 一个作业可以有多个学生提交 |

## 2. 建表 SQL

### 2.1 系统用户表（sys_user）

用于存储系统登录用户信息，包括管理员、教师等。

```sql
-- 系统用户表
CREATE TABLE `sys_user` (
    `id` BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    `username` VARCHAR(50) NOT NULL COMMENT '用户名，用于登录',
    `password` VARCHAR(255) NOT NULL COMMENT '密码，BCrypt加密存储',
    `real_name` VARCHAR(50) NOT NULL COMMENT '真实姓名',
    `role` TINYINT NOT NULL DEFAULT 3 COMMENT '角色：1-超级管理员，2-教务管理员，3-班主任，4-任课教师',
    `phone` VARCHAR(20) DEFAULT NULL COMMENT '联系电话',
    `email` VARCHAR(100) DEFAULT NULL COMMENT '邮箱地址',
    `avatar` VARCHAR(255) DEFAULT NULL COMMENT '头像URL',
    `login_fail_count` INT NOT NULL DEFAULT 0 COMMENT '连续登录失败次数',
    `locked_until` DATETIME DEFAULT NULL COMMENT '账户锁定截止时间，NULL表示未锁定',
    `last_login_time` DATETIME DEFAULT NULL COMMENT '最后登录时间',
    `status` TINYINT NOT NULL DEFAULT 1 COMMENT '状态：1-启用，0-禁用',
    `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    `is_deleted` TINYINT NOT NULL DEFAULT 0 COMMENT '逻辑删除标识：0-未删除，1-已删除'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='系统用户表';
```

**索引设计：**
- PRIMARY KEY (`id`)：主键索引
- UNIQUE KEY `uk_username` (`username`)：唯一索引，保证用户名唯一，用于登录查询
- KEY `idx_role` (`role`)：普通索引，按角色查询用户时使用
- KEY `idx_status` (`status`)：普通索引，查询启用/禁用用户时使用

### 2.2 班级表（class）

存储班级基本信息，每个班级有一个负责老师。

```sql
-- 班级表
CREATE TABLE `class` (
    `id` BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    `class_name` VARCHAR(30) NOT NULL COMMENT '班级名称，如"一年级一班"',
    `grade` TINYINT NOT NULL COMMENT '年级：1-12，对应小学到高中',
    `teacher_id` BIGINT NOT NULL COMMENT '负责老师ID，关联sys_user表',
    `classroom` VARCHAR(50) DEFAULT NULL COMMENT '教室位置',
    `max_students` INT NOT NULL DEFAULT 50 COMMENT '最大人数，范围10-100',
    `establish_date` DATE NOT NULL COMMENT '成立日期',
    `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    `is_deleted` TINYINT NOT NULL DEFAULT 0 COMMENT '逻辑删除标识：0-未删除，1-已删除'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='班级表';
```

**索引设计：**
- PRIMARY KEY (`id`)：主键索引
- UNIQUE KEY `uk_class_name` (`class_name`)：唯一索引，保证班级名称唯一
- KEY `idx_teacher_id` (`teacher_id`)：普通索引，查询某个老师负责的班级时使用
- KEY `idx_grade` (`grade`)：普通索引，按年级筛选班级时使用

## 3. 索引设计总结

### 3.1 索引设计原则

1. **主键索引**：每张表都有自增主键 `id`，用于唯一标识记录
2. **唯一索引**：用于保证业务唯一性约束（如学号、用户名等）
3. **外键索引**：所有关联其他表的字段都建立索引，提高JOIN查询性能
4. **查询索引**：根据高频查询场景建立索引（如按日期查询、按状态筛选等）
5. **联合索引**：对于经常一起查询的多个字段，建立联合索引提高效率

### 3.2 各表索引汇总

| 表名 | 索引数量 | 主要用途 |
|-----|---------|---------|
| sys_user | 4个 | 登录验证、角色查询、状态筛选 |
| class | 4个 | 班级名称唯一性、按老师查询、按年级筛选 |
| student | 5个 | 学号唯一性、按班级查询、按姓名搜索、按入学日期排序 |
| course_schedule | 5个 | 按班级查询课表、检查教师冲突、按时间段查询 |
| attendance | 5个 | 学生日期唯一性、按日期查询、按状态统计、按学生查询 |
| score | 6个 | 学生科目考试唯一性、按学生查询、按考试名称查询、按科目查询 |
| homework | 6个 | 按班级查询、按教师查询、按截止时间查询、按状态筛选 |
| homework_submission | 6个 | 作业学生唯一性、按作业查询、按学生查询、按时效性统计 |

### 3.3 性能优化建议

1. **避免过度索引**：每张表的索引数量控制在合理范围内（5-6个），过多索引会影响写入性能
2. **定期维护索引**：定期执行 `OPTIMIZE TABLE` 优化索引结构
3. **监控慢查询**：开启慢查询日志，分析并优化没有使用索引的查询
4. **合理使用覆盖索引**：对于只需要查询索引字段的场景，可以避免回表查询
5. **注意索引选择性**：对于选择性低的字段（如性别、状态），谨慎使用索引

## 4. 表关系总结

### 4.1 核心关系

1. **用户与班级**：一对多关系
   - 一个教师可以负责多个班级
   - 通过 `class.teacher_id` 关联 `sys_user.id`

2. **班级与学生**：一对多关系
   - 一个班级可以有多个学生
   - 一个学生只能属于一个班级
   - 通过 `student.class_id` 关联 `class.id`

3. **学生与考勤**：一对多关系
   - 一个学生可以有多条考勤记录
   - 通过 `attendance.student_id` 关联 `student.id`

4. **学生与成绩**：一对多关系
   - 一个学生可以有多条成绩记录
   - 通过 `score.student_id` 关联 `student.id`

5. **学生与作业提交**：一对多关系
   - 一个学生可以提交多次作业
   - 通过 `homework_submission.student_id` 关联 `student.id`

6. **班级与课程**：一对多关系
   - 一个班级可以有多条课程安排
   - 通过 `course_schedule.class_id` 关联 `class.id`

7. **班级与作业**：一对多关系
   - 一个班级可以接收多个作业
   - 通过 `homework.class_id` 关联 `class.id`

8. **作业与提交**：一对多关系
   - 一个作业可以有多个学生提交
   - 通过 `homework_submission.homework_id` 关联 `homework.id`

### 4.2 数据完整性约束

1. **外键约束**：虽然SQL中没有显式声明FOREIGN KEY，但应用层需要保证外键引用的完整性
2. **逻辑删除级联**：删除父记录时，需要检查是否有子记录依赖
   - 删除班级前，必须确保该班级下没有学生
   - 删除学生前，需要确认其考勤、成绩、作业提交记录的处理方式
3. **唯一性约束**：通过UNIQUE INDEX保证业务唯一性
   - 学号唯一
   - 用户名唯一
   - 班级名称唯一
   - 同一天同一学生只能有一条考勤记录
   - 同一学生同一科目同一考试只能有一条成绩记录
   - 同一学生对同一作业只能有一条提交记录
