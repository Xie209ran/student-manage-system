# 教学数字平台后台管理系统 - 后端开发计划

## 说明

- 本计划按执行顺序排列，前面的任务必须先完成
- 每个任务是小颗粒度的，一次对话可以完成
- 任务之间尽量无依赖，如有依赖已在前置任务中完成
- 开发顺序：基础配置 → 实体和Mapper → Controller/Service接口 → 认证拦截器

### 任务4：实现登录接口
**任务内容**：实现用户登录接口，包含密码验证和Token生成

**涉及文件**：
- `backend/src/main/java/com/edum/controller/AuthController.java`
- `backend/src/main/java/com/edum/service/UserService.java`
- `backend/src/main/java/com/edum/service/impl/UserServiceImpl.java`

**前置任务**：任务3

**【验收点】**
1. AuthController提供POST /api/auth/login接口
2. 接收username和password参数
3. 参数校验（不能为空，密码不少于6位）
4. 查询用户是否存在
5. 使用BCrypt验证密码
6. 检查账户是否被锁定
7. 登录失败次数累加
8. 连续失败5次后锁定30分钟
9. 登录成功生成JWT Token（有效期24小时）
10. 更新最后登录时间
11. 返回token和用户信息
12. 统一返回格式

**【验收标准】**
- 输入错误的用户名或密码，返回"用户名或密码错误"
- 连续失败5次后，返回"账户已锁定，请XX分钟后重试"
- 输入正确的用户名密码，返回token和用户信息
- Token格式正确，包含用户ID和过期时间
- 使用Postman测试接口，返回符合接口文档

---

### 任务5：实现退出登录和获取用户信息接口
**任务内容**：实现退出登录和获取当前用户信息接口

**涉及文件**：
- `backend/src/main/java/com/edum/controller/AuthController.java`（续）

**前置任务**：任务4

**【验收点】**
1. POST /api/auth/logout接口，使Token失效
2. GET /api/auth/current接口，从Token中解析用户ID，查询用户信息
3. 需要Token认证（通过拦截器或注解）
4. 返回用户详细信息（不包含密码）
5. 统一返回格式

**【验收标准】**
- 携带有效Token访问/current，返回用户信息
- Token无效或过期，返回401错误
- 调用logout后，原Token失效
- 使用Postman测试接口，返回符合接口文档

### 任务6：实现Token认证拦截器
**任务内容**：创建JWT Token认证拦截器，保护需要认证的接口

**涉及文件**：
- `backend/src/main/java/com/edum/config/JwtInterceptor.java`
- `backend/src/main/java/com/edum/config/WebConfig.java`
- `backend/src/main/java/com/edum/util/JwtUtil.java`

**前置任务**：任务4

**【验收点】**
1. JwtUtil提供生成Token和解析Token的方法
2. JwtInterceptor拦截请求，验证Token
3. 从Token中解析用户ID，放入request attribute
4. WebConfig配置拦截器，排除登录接口
5. Token无效或过期返回401
6. 支持从Authorization header中读取Bearer Token

**【验收标准】**
- 不带Token访问受保护接口，返回401
- 带有效Token访问，能正常访问
- Token过期后访问，返回401
- 能从request中获取当前用户ID
