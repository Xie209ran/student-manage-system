# 教学数字平台后台管理系统 - 接口文档

## 1. 接口说明

### 1.1 基础信息
- **Base URL**: `/api`
- **数据格式**: JSON
- **字符编码**: UTF-8
- **认证方式**: Token（JWT）

### 1.2 统一返回格式

所有接口均使用统一的返回格式：

**成功返回**:
```json
{
  "code": 200,
  "message": "success",
  "data": {...}
}
```

**失败返回**:
```json
{
  "code": 400,
  "message": "错误信息描述",
  "data": null
}
```

**返回码说明：**

| 返回码 | 说明 |
|-------|------|
| 200 | 请求成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或Token失效 |
| 403 | 权限不足 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

### 1.3 分页规范

所有列表查询接口的分页参数和返回格式统一如下：

**请求参数**:
- `pageNum`: 第几页（从1开始），默认1
- `pageSize`: 每页多少条，默认10

**返回格式**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total": 100,
    "list": [...]
  }
}
```

### 1.4 请求头要求

所有需要认证的接口都需要在请求头中携带Token：

```
Authorization: Bearer {token}
Content-Type: application/json
```

## 2. 登录模块接口

### 2.1 用户登录

**接口地址**: `POST /api/auth/login`

**接口描述**: 用户通过用户名和密码登录系统

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 | 校验规则 |
|-------|------|------|------|---------|
| username | String | 是 | 用户名 | 不能为空 |
| password | String | 是 | 密码 | 不能为空，长度不少于6位 |

**请求示例**:

```json
{
  "username": "admin",
  "password": "123456"
}
```

**返回示例**:

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "userInfo": {
      "id": 1,
      "username": "admin",
      "realName": "管理员",
      "role": 1,
      "avatar": "http://example.com/avatar.jpg"
    }
  }
}
```

**业务规则**:
- 连续登录失败5次后，账户锁定30分钟
- 登录成功后生成Token，有效期24小时
- 密码使用BCrypt加密验证

---

### 2.2 退出登录

**接口地址**: `POST /api/auth/logout`

**接口描述**: 用户退出登录，使Token失效

**请求参数**: 无

**返回示例**:

```json
{
  "code": 200,
  "message": "success",
  "data": null
}
```

---

### 2.3 获取当前用户信息

**接口地址**: `GET /api/auth/current`

**接口描述**: 获取当前登录用户的详细信息

**请求参数**: 无（需要从Token中解析用户ID）

**返回示例**:

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "username": "admin",
    "realName": "管理员",
    "role": 1,
    "phone": "13800138000",
    "email": "admin@example.com",
    "avatar": "http://example.com/avatar.jpg",
    "status": 1
  }
}
```

**接口描述**: 获取仪表盘的统计数字卡片数据

    "totalStudents": 1250,
    "totalClasses": 35,
    "todayAttendanceRate": 95.6,
    "pendingHomeworkCount": 128

**字段说明**:

| 字段名 | 类型 | 说明 |

## 4. 学生管理模块接口

### 4.1 分页查询学生列表

**接口地址**: `GET /api/students`

**接口描述**: 分页查询学生列表，支持按姓名搜索和按班级筛选

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 | 位置 |
|-------|------|------|------|------|
| pageNum | Integer | 否 | 页码，默认1 | URL |
| pageSize | Integer | 否 | 每页数量，默认10，可选10/20/50 | URL |
| name | String | 否 | 学生姓名，支持模糊搜索 | URL |
| classId | Long | 否 | 班级ID，按班级筛选 | URL |
| sortBy | String | 否 | 排序字段，默认enrollment_date | URL |
| sortOrder | String | 否 | 排序方式，asc/desc，默认desc | URL |

**请求示例**:

```
GET /api/students?pageNum=1&pageSize=10&name=张&classId=1&sortBy=enrollment_date&sortOrder=desc
```

**返回示例**:

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total": 125,
    "list": [
      {
        "id": 1,
        "studentNo": "20240101",
        "name": "张三",
        "gender": 1,
        "birthDate": "2010-05-15",
        "classId": 1,
        "className": "一年级一班",
        "phone": "13800138000",
        "address": "北京市朝阳区xxx",
        "enrollmentDate": "2024-09-01"

}
```

**失败返回示例**:

### 4.3 新增学生

**接口地址**: `POST /api/students`

**接口描述**: 新增学生信息

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 | 校验规则 |
|-------|------|------|------|---------|
| studentNo | String | 是 | 学号 | 唯一，8-12位数字或字母 |
| name | String | 是 | 姓名 | 2-20个字符 |
| gender | Integer | 是 | 性别 | 1-男，2-女 |
| birthDate | String | 是 | 出生日期 | 格式YYYY-MM-DD，不能晚于今天 |
| classId | Long | 是 | 班级ID | 必须存在且未删除 |
| phone | String | 否 | 联系电话 | 11位手机号格式 |
| address | String | 否 | 家庭住址 | 最多200个字符 |
| enrollmentDate | String | 是 | 入学日期 | 格式YYYY-MM-DD |

**请求示例**:

```json
{
  "studentNo": "20240101",
  "name": "张三",
  "gender": 1,
  "birthDate": "2010-05-15",
  "classId": 1,
  "phone": "13800138000",
  "address": "北京市朝阳区xxx",
  "enrollmentDate": "2024-09-01"
}
```

**返回示例**:

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1
  }
}
```

**特殊情况**:
- 学号重复时返回错误："该学号已存在"
- 班级不存在时返回错误："班级不存在"
- 班级满员时返回错误："该班级已满员"

### 6.2 添加课程

**接口地址**: `POST /api/courses/schedule`

**接口描述**: 为班级添加课程安排

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 | 校验规则 |
|-------|------|------|------|---------|
| classId | Long | 是 | 班级ID | 必须存在 |
| dayOfWeek | Integer | 是 | 星期 | 1-7 |
| period | Integer | 是 | 节次 | 1-8 |
| subject | String | 是 | 课程科目 | 从预设科目中选择 |
| teacherId | Long | 是 | 任课教师ID | 必须存在 |
| classroom | String | 否 | 教室 | 最多50个字符 |
| startDate | String | 是 | 开始日期 | 格式YYYY-MM-DD |
| endDate | String | 是 | 结束日期 | 必须晚于开始日期 |

**请求示例**:

```json
{
  "classId": 1,
  "dayOfWeek": 1,
  "period": 1,
  "subject": "语文",
  "teacherId": 2,
  "classroom": "A101",
  "startDate": "2026-05-18",
  "endDate": "2026-07-18"
}
```

**返回示例**:

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1
  }
}
```

**业务规则**:
- 检测时间冲突：同一班级在同一时间段只能有一门课程
- 检测教师冲突：同一教师在同一时间段只能在一个班级上课
- 冲突时返回错误："张三老师在周一第1节已有课程安排"

## 7. 考勤打卡模块接口

### 7.1 批量打卡

**接口地址**: `POST /api/attendance/batch`

**接口描述**: 按班级批量录入学生出勤状态

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| classId | Long | 是 | 班级ID |
| attendanceDate | String | 是 | 考勤日期，格式YYYY-MM-DD |
| records | Array | 是 | 考勤记录列表 |

**records数组元素**:

| 参数名 | 类型 | 必填 | 说明 |
|-------|------|------|------|
| studentId | Long | 是 | 学生ID |
| status | Integer | 是 | 出勤状态：1-出勤，2-迟到，3-请假，4-缺勤 |
| remark | String | 否 | 备注 |

**请求示例**:

## 12. 接口设计规范总结

### 12.1 统一规范

为确保接口的一致性和易用性，所有接口遵循以下设计规范：

#### 12.1.1 接口地址规范
- 所有接口地址以 `/api/` 开头
- 使用RESTful风格设计
- 资源名称使用复数形式（如 `/api/students`）

#### 12.1.2 HTTP方法规范
- **GET**: 查询数据（不修改服务器状态）
- **POST**: 新增数据
- **PUT**: 修改数据
- **DELETE**: 删除数据

#### 12.1.3 统一返回格式

**成功返回**:
```json
{
  "code": 200,
  "message": "success",
  "data": {...}
}
```

**失败返回**:
```json
{
  "code": 400,
  "message": "错误信息描述",
  "data": null
}
```

#### 12.1.4 分页规范

所有列表查询接口统一使用以下分页参数和返回格式：

**请求参数**（放在URL中）:
- `pageNum`: 第几页（从1开始），默认1
- `pageSize`: 每页多少条，默认10

**返回格式**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total": 100,
    "list": [...]
  }
}
```

**注意**: 
- 只返回 `total` 和 `list` 两个字段
- 不返回 `pageNum`、`pageSize`、`pages` 等额外字段
- 前端根据 `total` 和 `pageSize` 自行计算总页数

#### 12.1.5 参数位置规范

- **URL参数**: GET请求的查询参数、路径参数
- **请求体**: POST/PUT请求的业务数据（JSON格式）
- **请求头**: Token认证信息

#### 12.1.6 参数命名规范

- 使用驼峰命名法（camelCase）
- 布尔值以 `is` 开头（如 `isDeleted`）
- ID字段以 `Id` 结尾（如 `classId`、`studentId`）

### 12.2 常见错误码

| 错误码 | 说明 | 示例场景 |
|-------|------|----------|
| 200 | 成功 | - |
| 400 | 请求参数错误 | 参数校验失败、格式错误 |
| 401 | 未登录或Token失效 | Token过期、Token无效 |
| 403 | 权限不足 | 访问无权访问的资源 |
| 404 | 资源不存在 | 查询的数据不存在 |
| 500 | 服务器内部错误 | 系统异常、数据库错误 |

### 12.3 业务错误提示

对于业务逻辑错误，在 `message` 字段中返回明确的中文提示：

- "该学号已存在"
- "班级名称已存在"
- "该班级下有X名学生，请先转移或删除学生"
- "该学生有X个待批改作业，请先处理"
- "张三老师在周一第1节已有课程安排"
- "用户名或密码错误"
- "账户已锁定，请XX分钟后重试"
