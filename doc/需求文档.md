# 教学数字平台后台管理系统 - 软件需求说明书

## 1. 引言

### 1.1 项目背景
教学数字平台后台管理系统是一个面向学校和教育机构的综合管理平台，旨在帮助教师和管理员高效地管理学生信息、班级事务、课程安排、考勤记录、成绩管理和作业批改等教学工作。

### 1.2 系统目标
- 提供统一的教学数据管理平台
- 简化日常教学管理工作流程
- 实现数据的可视化展示和分析
- 提高教学管理效率和准确性

### 1.3 适用范围
本系统适用于中小学及培训机构的教学管理人员、班主任和任课教师使用。

## 2. 功能模块详细说明

### 2.1 登录模块

#### 2.1.1 功能描述
提供用户身份验证功能，只有通过用户名和密码验证的用户才能访问系统。

#### 2.1.2 用户操作
- 输入用户名
- 输入密码
- 点击登录按钮
- 忘记密码（可选功能）

#### 2.1.3 业务规则
| 规则编号 | 规则描述 |
|---------|---------|
| R001 | 用户名和密码不能为空 |
| R002 | 密码长度不少于6位 |
| R003 | 连续登录失败5次后，账户锁定30分钟 |
| R004 | 登录成功后生成会话令牌，有效期为24小时 |
| R005 | 密码在数据库中必须加密存储（使用BCrypt等加密算法） |

#### 2.1.4 特殊情况处理
- 用户名不存在时，提示"用户名或密码错误"（不区分具体错误，防止暴力破解）
- 账户被锁定时，明确提示锁定剩余时间
- 会话过期时，自动跳转到登录页面
- 网络异常时，显示友好的错误提示

### 2.2 仪表盘模块

#### 2.2.1 功能描述
系统首页，通过图表和统计卡片直观展示教学数据概况，帮助管理员快速了解整体情况。

#### 2.2.2 用户操作
- 查看统计数据卡片
- 查看各班级学生人数分布图
- 查看近7天出勤趋势图
- 刷新数据（可选）

#### 2.2.3 业务规则

**统计数字卡片：**
| 卡片名称 | 数据来源 | 更新频率 |
|---------|---------|---------|
| 学生总数 | 学生表中is_deleted=0的记录数 | 实时 |
| 班级总数 | 班级表中is_deleted=0的记录数 | 实时 |
| 今日出勤率 | 今日已打卡学生中出勤和迟到人数 / 应到学生数 × 100% | 实时 |
| 待批改作业数 | 作业状态为"已提交未批改"的数量 | 实时 |

**各班级学生人数分布图：**
- 图表类型：柱状图或饼图
- X轴：班级名称
- Y轴：学生人数
- 只显示未删除的班级
- 按学生人数降序排列

**近7天出勤趋势图：**
- 图表类型：折线图
- X轴：最近7天的日期
- Y轴：出勤率百分比
- 出勤率 = (出勤人数 + 迟到人数) / 应到人数 × 100%
- 如果某天没有考勤记录，该天数据显示为0

#### 2.2.4 特殊情况处理
- 如果没有班级数据，显示"暂无数据"提示
- 如果某班级没有学生，该班级在图表中显示为0
- 数据加载失败时，显示重试按钮
- 图表数据过多时，支持横向滚动或分页展示

### 2.3 学生管理模块

#### 2.3.1 功能描述
对学生信息进行全生命周期管理，包括新增、编辑、删除、查询等操作。

#### 2.3.2 用户操作
- **新增学生**：填写学生基本信息并保存
- **编辑学生**：修改已有学生的信息
- **删除学生**：删除学生记录（逻辑删除）
- **查询学生**：浏览学生列表
- **搜索学生**：按姓名模糊搜索
- **筛选学生**：按班级筛选
- **翻页浏览**：支持分页查看学生列表

#### 2.3.3 业务规则

**学生信息字段：**
| 字段名称 | 是否必填 | 校验规则 |
|---------|---------|---------|
| 学号 | 是 | 唯一，8-12位数字或字母 |
| 姓名 | 是 | 2-20个字符，支持中文 |
| 性别 | 是 | 男/女 |
| 出生日期 | 是 | 日期格式YYYY-MM-DD，不能晚于今天 |
| 所属班级 | 是 | 必须选择已存在的班级 |
| 联系电话 | 否 | 11位手机号格式 |
| 家庭住址 | 否 | 最多200个字符 |
| 入学日期 | 是 | 日期格式YYYY-MM-DD |

**列表展示规则：**
- 每页默认显示10条记录
- 可选择每页显示10/20/50条
- 默认按入学日期降序排列
- 显示字段：学号、姓名、性别、班级、联系电话、操作

**搜索和筛选规则：**
- 姓名搜索支持模糊匹配
- 班级筛选为下拉选择，只显示未删除的班级
- 搜索和筛选可以同时使用
- 清空搜索条件后恢复完整列表

#### 2.3.4 特殊情况处理
- **学号重复**：新增或修改时检查学号唯一性，重复则提示"该学号已存在"
- **删除学生前检查**：
  - 检查该学生是否有未批改的作业，如有则提示"该学生有X个待批改作业，请先处理"
  - 检查该学生是否有考勤记录，如有则二次确认"该学生有考勤记录，删除后不可恢复，是否继续？"
- **班级不存在**：如果学生所属班级已被删除，在学生列表中显示"班级已删除"标记
- **批量操作**：支持批量删除学生，删除前需要二次确认
- **导入导出**：支持Excel批量导入学生信息，支持导出学生列表

### 2.4 班级管理模块

#### 2.4.1 功能描述
对班级信息进行管理，包括创建班级、分配负责老师、维护班级信息等。

#### 2.4.2 用户操作
- **新增班级**：创建新班级并指定负责老师
- **编辑班级**：修改班级信息和负责老师
- **删除班级**：删除班级记录（逻辑删除）
- **查询班级**：浏览班级列表
- **查看班级详情**：查看班级详细信息和学生列表

#### 2.4.3 业务规则

**班级信息字段：**
| 字段名称 | 是否必填 | 校验规则 |
|---------|---------|---------|
| 班级名称 | 是 | 唯一，2-30个字符，如"一年级一班" |
| 年级 | 是 | 1-12（小学到高中） |
| 负责老师 | 是 | 必须选择已存在的教师 |
| 教室位置 | 否 | 最多50个字符 |
| 最大人数 | 是 | 正整数，范围10-100 |
| 成立日期 | 是 | 日期格式YYYY-MM-DD |

**列表展示规则：**
- 每页默认显示10条记录
- 显示字段：班级名称、年级、负责老师、学生人数/最大人数、成立日期、操作
- 学生人数实时统计，显示格式如"35/50"
- 当学生人数达到最大人数的90%时，显示预警标记

#### 2.4.4 特殊情况处理
- **班级名称重复**：新增或修改时检查班级名称唯一性
- **删除班级前检查**：
  - 检查班级下是否有学生，如有学生则禁止删除，提示"该班级下有X名学生，请先转移或删除学生"
  - 检查班级是否有课程安排，如有则提示"该班级有课程安排，请先取消课程"
- **负责老师变更**：
  - 更换负责老师时，记录变更历史
  - 原负责老师不再拥有该班级的管理权限
- **班级满员**：当班级学生人数达到最大人数时，新增学生时提示"该班级已满员"
- **年级限制**：一个年级可以有多个班级，但班级名称必须在整个系统中唯一

### 2.5 课程排课模块

#### 2.5.1 功能描述
为各个班级安排每周的课程表，以表格形式直观展示课程安排。

#### 2.5.2 用户操作
- **查看课表**：按班级查看周课表
- **添加课程**：为班级添加课程安排
- **编辑课程**：修改已有课程的时间、科目或教师
- **删除课程**：取消某个课程安排
- **切换班级**：在不同班级课表之间切换
- **打印课表**：导出或打印课表

#### 2.5.3 业务规则

**课程信息字段：**
| 字段名称 | 是否必填 | 校验规则 |
|---------|---------|---------|
| 班级 | 是 | 必须选择已存在的班级 |
| 星期 | 是 | 周一至周日 |
| 节次 | 是 | 第1节-第8节（可根据学校实际情况调整） |
| 课程科目 | 是 | 从预设科目中选择（语文、数学、英语等） |
| 任课教师 | 是 | 必须选择已存在的教师 |
| 教室 | 否 | 最多50个字符 |
| 开始日期 | 是 | 课表生效日期 |
| 结束日期 | 是 | 课表失效日期，必须晚于开始日期 |

**课表展示规则：**
- 表格形式：行表示节次（第1节至第8节），列表示星期（周一至周五）
- 每个单元格显示：课程科目、任课教师、教室（如果有）
- 不同科目用不同颜色区分
- 空白单元格表示该时间段无课程

**排课冲突检测：**
- 同一班级在同一时间段只能有一门课程
- 同一教师在同一时间段只能在一个班级上课
- 同一教室在同一时间段只能有一个班级使用（如果启用了教室管理）

#### 2.5.4 特殊情况处理
- **时间冲突**：添加或修改课程时，如果检测到冲突，明确提示冲突详情（如"张三老师在周一第1节已有课程安排"）
- **教师不存在**：如果课程的任课教师已被删除，在课表中显示"教师已离职"标记
- **跨周期课表**：支持设置课表的有效期，过期后自动失效
- **临时调课**：支持单次调课，不影响常规课表
- **节假日处理**：可以标记节假日，节假日当天的课程自动暂停
- **复制课表**：支持将一个班级的课表复制到另一个班级（需手动调整教师）

### 2.6 考勤打卡模块

#### 2.6.1 功能描述
记录学生每日的出勤情况，支持多种出勤状态，方便统计和分析学生出勤率。

#### 2.6.2 用户操作
- **批量打卡**：按班级批量录入学生出勤状态
- **单个打卡**：为单个学生记录出勤状态
- **查看考勤**：按日期和班级查看考勤记录
- **修改考勤**：修正错误的考勤记录
- **统计查询**：按学生或班级统计出勤情况
- **导出考勤**：导出考勤记录为Excel

#### 2.6.3 业务规则

**出勤状态定义：**
| 状态 | 说明 | 是否计为出勤 |
|-----|------|------------|
| 出勤 | 正常按时到校 | 是 |
| 迟到 | 超过规定时间到校 | 是（但单独统计） |
| 请假 | 提前申请并获得批准的缺勤 | 否 |
| 缺勤 | 未经请假的缺勤 | 否 |

**考勤记录字段：**
| 字段名称 | 是否必填 | 校验规则 |
|---------|---------|---------|
| 学生 | 是 | 必须选择已存在且未删除的学生 |
| 考勤日期 | 是 | 日期格式YYYY-MM-DD，不能晚于今天 |
| 出勤状态 | 是 | 出勤/迟到/请假/缺勤 |
| 备注 | 否 | 最多200个字符，用于说明特殊情况 |
| 打卡人 | 是 | 自动记录当前操作用户 |
| 打卡时间 | 是 | 自动记录操作时间 |

**批量打卡规则：**
- 选择班级和日期后，列出该班级所有学生
- 默认所有学生状态为"出勤"
- 可以逐个修改学生状态
- 支持一键设置所有学生为某种状态
- 提交前需要确认

**统计规则：**
- 个人出勤率 = (出勤天数 + 迟到天数) / 应到天数 × 100%
- 班级出勤率 = 班级所有学生出勤率的平均值
- 应到天数 = 考勤日期范围内的总天数 - 周末 - 法定节假日
- 统计时可以按日、周、月、学期等维度查看

#### 2.6.4 特殊情况处理
- **重复打卡**：同一天同一学生只能有一条考勤记录，再次打卡时提示"该学生今日已打卡，是否修改？"
- **补卡处理**：允许补录之前日期的考勤，但需要标注为"补录"
- **学生转班**：如果学生在考勤日期之后转班，考勤记录保持不变，归属原班级
- **批量修改**：支持批量修改某个日期范围内某个班级的考勤状态（如集体活动）
- **请假关联**：如果学生有已批准的请假申请，打卡时自动默认为"请假"状态
- **异常提醒**：连续缺勤3天以上的学生，在仪表盘显示预警
- **数据一致性**：删除学生时，其考勤记录保留但标记学生为"已删除"

### 2.7 成绩管理模块

#### 2.7.1 功能描述
管理学生的各科考试成绩，支持成绩录入、查询、统计和分析。

#### 2.7.2 用户操作
- **录入成绩**：为班级学生录入某科目的考试成绩
- **编辑成绩**：修改已录入的成绩
- **删除成绩**：删除错误的成绩记录
- **查询成绩**：按学生、班级、科目、考试名称等条件查询
- **成绩统计**：查看班级平均分、最高分、最低分等统计数据
- **导出成绩**：导出成绩单为Excel或PDF
- **成绩分析**：查看成绩分布图、进步/退步趋势

#### 2.7.3 业务规则

**成绩记录字段：**
| 字段名称 | 是否必填 | 校验规则 |
|---------|---------|---------|
| 学生 | 是 | 必须选择已存在且未删除的学生 |
| 科目 | 是 | 从预设科目中选择 |
| 考试名称 | 是 | 如"期中考试"、"期末考试"、"月考1"等 |
| 考试成绩 | 是 | 数值型，范围0-150（可根据科目调整） |
| 满分值 | 是 | 默认100或150，可自定义 |
| 考试日期 | 是 | 日期格式YYYY-MM-DD |
| 任课教师 | 是 | 自动记录当前操作用户或选择的教师 |
| 备注 | 否 | 最多200个字符 |

**成绩录入规则：**
- 可以按班级批量录入某次考试的所有学生成绩
- 录入界面以表格形式展示，类似Excel
- 支持键盘快速录入（Tab键切换下一个输入框）
- 录入完成后统一保存，部分失败不影响其他记录

**成绩查询规则：**
- 可以按以下条件组合查询：
  - 学生姓名或学号
  - 班级
  - 科目
  - 考试名称
  - 考试时间范围
- 查询结果可以按成绩高低排序
- 显示字段：学生、班级、科目、考试名称、成绩、满分、考试日期、排名

**统计规则：**
- 班级平均分：某次考试班级所有学生成绩的平均值
- 最高分/最低分：某次考试的最高和最低成绩
- 及格率：成绩 >= 60分（或满分的60%）的学生比例
- 优秀率：成绩 >= 90分（或满分的90%）的学生比例
- 排名：可以按班级排名或年级排名

#### 2.7.4 特殊情况处理
- **重复录入**：同一学生同一科目同一考试名称只能有一条成绩记录，再次录入时提示"该学生此考试成绩已存在，是否覆盖？"
- **成绩修改历史**：每次修改成绩都记录修改人和修改时间，可查看修改历史
- **学生转班**：学生转班后，历史成绩保持不变，归属原班级
- **缺考处理**：如果学生缺考，成绩记为0分，但可以标注为"缺考"，统计时可以选择是否包含缺考学生
- **成绩审核**：成绩录入后可以设置为"待审核"状态，审核后不可随意修改
- **隐私保护**：普通教师只能查看自己任教科目的成绩，班主任可以查看本班所有科目成绩
- **成绩分布图**：用柱状图展示各分数段的人数分布（如0-59、60-69、70-79、80-89、90-100）
- **进步分析**：对比同一学生多次考试的成绩，显示进步或退步幅度

### 2.8 作业管理模块

#### 2.8.1 功能描述
管理作业的发布、提交、批改全流程，方便教师跟踪学生作业完成情况。

#### 2.8.2 用户操作

**教师端操作：**
- **发布作业**：创建新作业并布置给班级
- **查看提交情况**：查看哪些学生已提交、哪些未提交
- **批改作业**：为学生作业打分和写评语
- **下载作业**：下载学生提交的作业文件
- **统计分析**：查看作业完成率、平均分等统计

**学生端操作（如果有学生端）：**
- **查看作业**：查看老师发布的作业
- **提交作业**：上传作业文件或在线作答
- **查看批改**：查看老师的评分和评语

#### 2.8.3 业务规则

**作业信息字段：**
| 字段名称 | 是否必填 | 校验规则 |
|---------|---------|---------|
| 作业标题 | 是 | 2-100个字符 |
| 作业内容 | 是 | 最多2000个字符，支持富文本 |
| 布置班级 | 是 | 至少选择一个班级 |
| 布置科目 | 是 | 从预设科目中选择 |
| 布置教师 | 是 | 自动记录当前操作用户 |
| 布置时间 | 是 | 自动记录当前时间 |
| 截止时间 | 是 | 必须晚于布置时间 |
| 满分分值 | 是 | 正整数，默认100 |
| 附件 | 否 | 支持上传作业相关文件，单个文件不超过10MB |

**作业状态定义：**
| 状态 | 说明 |
|-----|------|
| 进行中 | 未到截止时间，学生可以提交 |
| 已结束 | 已过截止时间，不能再提交 |
| 已批改 | 教师已完成批改 |

**学生提交记录字段：**
| 字段名称 | 是否必填 | 校验规则 |
|---------|---------|---------|
| 学生 | 是 | 必须选择已存在且未删除的学生 |
| 作业 | 是 | 必须选择已存在的作业 |
| 提交时间 | 是 | 自动记录提交时间 |
| 提交内容 | 否 | 文本内容或文件路径 |
| 提交文件 | 否 | 支持上传文件，单个文件不超过10MB |
| 是否迟交 | 是 | 根据提交时间和截止时间自动判断 |
| 得分 | 否 | 教师批改后填写，范围0-满分分值 |
| 评语 | 否 | 最多500个字符 |
| 批改教师 | 否 | 自动记录批改人 |
| 批改时间 | 否 | 自动记录批改时间 |

**作业提交流程：**
1. 教师发布作业，选择班级和截止时间
2. 学生查看作业并在规定时间内提交
3. 教师在截止时间后批改作业
4. 学生查看批改结果

**统计规则：**
- 完成率 = 已提交人数 / 班级总人数 × 100%
- 按时完成率 = 按时提交人数 / 已提交人数 × 100%
- 平均分 = 所有已批改作业得分的平均值
- 未提交名单：列出所有未提交作业的学生

#### 2.8.4 特殊情况处理
- **迟交处理**：超过截止时间提交的作业标记为"迟交"，但仍可提交和批改
- **重新提交**：在截止时间前，学生可以多次提交，只保留最后一次提交记录
- **批量批改**：支持批量给分（如全部给满分），然后逐个修改
- **作业延期**：教师可以延长作业截止时间，已提交的学生不受影响
- **撤回作业**：在没有任何学生提交前，教师可以删除作业
- **有提交后删除**：如果已有学生提交，删除作业时需要二次确认，并提示"已有X名学生提交，删除后不可恢复"
- **文件预览**：支持常见文件格式的在线预览（PDF、图片等）
- **提醒功能**：截止前24小时和1小时，向未提交的学生发送提醒（如果有消息通知功能）
- **抄袭检测**：未来可扩展功能，检测学生作业的相似度

## 3. 通用业务规则

### 3.1 数据权限规则
| 角色 | 权限范围 |
|-----|---------|
| 超级管理员 | 可以访问所有功能和数据 |
| 教务管理员 | 可以管理学生、班级、课程，查看所有成绩和作业 |
| 班主任 | 可以管理本班学生，查看本班所有科目的成绩和作业 |
| 任课教师 | 只能查看和录入自己任教科目的成绩，只能布置和批改自己科目的作业 |

### 3.2 逻辑删除规则
- 所有删除操作都是逻辑删除，将is_deleted字段设置为1
- 查询时默认只查询is_deleted=0的记录
- 删除关联数据时需要检查依赖关系
- 已删除的数据可以在回收站中查看（可选功能）

### 3.3 数据校验规则
- 前端进行基础格式校验（必填、格式、长度等）
- 后端进行业务逻辑校验（唯一性、关联性、权限等）
- 校验失败时返回明确的错误信息
- 所有接口参数都必须经过校验

### 3.4 并发控制规则
- 同一用户不能在多处同时登录（可选）
- 重要数据修改时采用乐观锁机制
- 批量操作时保证事务一致性

- 页面加载时间不超过3秒

- 提供清晰的操作提示和错误信息
